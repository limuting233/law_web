## 1. 项目背景

本项目定位为面向普通用户、中小企业和法律服务供给方的综合法律服务网站。网站首期必须提供法律相关新闻展示、在线法律咨询、法律法规查询三个核心功能，并在此基础上逐步扩展法律知识、律师/机构展示、案例检索、文书模板、用户中心和运营后台能力。

法律服务类网站的用户需求通常不是单点需求，而是从“了解法律问题”到“查询依据”再到“获得咨询/委托服务”的连续链路。竞品调研显示：

- 公共法律服务平台通常围绕“咨询、办事、查询、机构地图、法援/调解/公证等服务入口”设计。
- 商业法律平台通常围绕“在线问律师、找律师、法律知识内容、电话/图文咨询、付费转化”设计。
- 专业法律检索平台通常围绕“法律法规、案例、裁判文书、关联法条、专题库、更新提醒和高级检索”设计。

因此，本项目不应只是一个新闻站或咨询表单站，而应设计成“内容获客 + 法规检索 + 咨询转化 + 后台运营”的综合平台。

## 2. 产品目标

### 2.1 业务目标

1. 建立可信的法律内容入口，提升网站搜索引擎收录和自然流量。
2. 为用户提供基础法律问题的自助查询、法规检索和在线咨询入口。
3. 为平台后续扩展律师入驻、付费咨询、企业法律服务、文书下载等商业化能力预留结构。
4. 通过管理系统支撑内容发布、咨询处理、用户管理、法规维护和运营配置。

### 2.2 用户目标

1. 普通个人用户可以快速了解法律新闻、法律知识，并提交法律问题。
2. 企业用户可以查询法规政策、获取合规知识，并发起专项咨询。
3. 平台运营人员可以高效维护新闻、法规、咨询、专题和首页配置。
4. 律师或法律服务人员可以查看分配给自己的咨询并进行回复。

### 2.3 技术目标

1. 前端官网使用 Vue 3，管理系统使用 Vue 3。
2. 后端服务使用 FastAPI，提供统一 REST API。
3. 支持前后端分离、权限控制、内容管理、搜索查询和后续可扩展的支付/IM/AI 能力。

## 3. 竞品与市场需求分析

### 3.1 参考平台

| 类型 | 平台 | 可参考能力 | 对本项目启发 |
| --- | --- | --- | --- |
| 公共服务 | 中国法律服务网 / 12348 | 公共法律咨询、法律法规、案例库、服务机构、法律地图等 | 需要覆盖“问、查、找服务”链路 |
| 公共服务 | 地方公共法律服务网 | 在线咨询、办事服务、法援/调解/公证入口 | 后续可扩展地域服务和机构入口 |
| 商业咨询 | 华律网 | 问律师、找律师、法律知识、专题内容、咨询转化 | 咨询入口必须高频、清晰、低门槛 |
| 商业咨询 | 找法网 | 法律咨询、找律师、法律知识、法规/案例相关内容 | 内容分类和 SEO 能力很重要 |
| 商业咨询 | 律图 | 法律知识、法律咨询、律师匹配、文书/计算工具 | 可扩展工具型功能提升留存 |
| 专业检索 | 北大法宝 | 法律法规、司法案例、专题库、高级检索 | 法规查询需要分类、筛选、更新、关联 |
| 专业检索 | 威科先行 | 法规、案例、实务内容、专业检索 | 中长期可做专业会员或企业服务 |
| 专业检索 | 法信 | 法律法规、案例、司法观点、类案检索 | 可扩展案例库和法条关联能力 |

### 3.2 市场需求提炼

1. 用户需要低门槛提问：大部分个人用户无法准确描述法律关系，需要通过分类、引导问题和示例降低提问成本。
2. 用户需要可信内容：法律新闻、政策解读和知识文章要有来源、发布时间、作者/编辑和审核状态。
3. 用户需要快速找到法律依据：法规查询不能只是列表，需要支持关键词、效力层级、地区、发布机关、时效性等筛选。
4. 用户需要从内容自然转向咨询：新闻、法规、知识文章详情页都应有相关咨询入口。
5. 企业用户更关注合规和效率：需要劳动用工、合同、公司治理、知识产权、数据合规等专题。
6. 平台运营需要后台工具：内容审核、咨询分派、法规维护、首页配置、数据统计是必须能力。
7. 法律服务需要合规边界：在线内容和智能回复应明确“不构成正式法律意见”，涉及律师服务需展示资质与执业信息。

## 4. 产品定位

### 4.1 一句话定位

一个面向个人和企业用户的法律资讯、法规查询与在线咨询综合服务平台。

### 4.2 核心价值

1. 对普通用户：看得懂、查得到、问得出。
2. 对企业用户：能检索、能咨询、能沉淀合规知识。
3. 对运营人员：可发布、可审核、可分派、可统计。
4. 对律师/服务人员：可接收咨询、可回复、可沉淀案例。

### 4.3 差异化方向

1. 将新闻、法规、知识和咨询打通，而不是孤立展示。
2. 用专题化组织内容，例如婚姻家事、劳动争议、交通事故、合同纠纷、公司法务。
3. 在法规详情页提供关联知识、相关案例、常见咨询和咨询入口。
4. 为后续 AI 法律助手、智能文书、律师入驻和企业会员预留能力。

## 7. 功能需求

### 7.1 首页

#### 功能说明

首页承担内容推荐、法规检索入口和咨询转化入口。

#### 页面模块

1. 顶部导航和全站搜索。
2. 首屏核心区：搜索框、热门法律问题分类、立即咨询按钮。
3. 法律新闻推荐：最新、热点、政策解读。
4. 法律法规快捷查询：关键词搜索、热门法规入口。
5. 在线咨询入口：选择问题类型、填写一句话问题。
6. 法律知识分类：婚姻家事、劳动工伤、交通事故、合同纠纷、房产纠纷、公司法务、刑事辩护、知识产权。
7. 热门案例解析。
8. 文书模板推荐。
9. 律师/机构推荐。
10. 友情链接和合规声明。

#### 验收标准

1. 用户可从首页进入新闻、法规查询和在线咨询。
2. 首页推荐内容可由管理后台配置。
3. 首页搜索支持跳转到统一搜索结果页。

### 7.2 法律相关新闻展示

#### 功能说明

展示法律、司法、政策、社会热点相关资讯，并支持分类、搜索、推荐和详情阅读。

#### 功能点

1. 新闻列表：按分类、发布时间、热度、推荐状态展示。
2. 新闻分类：法治新闻、政策解读、司法动态、企业合规、民生法律、案例热点。
3. 新闻详情：标题、摘要、封面、正文、来源、作者、发布时间、浏览量、标签。
4. 相关新闻推荐：根据分类、标签和关键词推荐。
5. 分享能力：复制链接，预留微信/微博分享。
6. SEO 信息：支持标题、描述、关键词配置。
7. 内容状态：草稿、待审核、已发布、已下线。

#### 验收标准

1. 用户可按分类浏览新闻。
2. 用户可通过关键词搜索新闻。
3. 新闻详情页展示来源和发布时间。
4. 后台可以新增、编辑、发布、下线新闻。

### 7.3 在线法律咨询

#### 功能说明

用户可以提交法律问题，平台可通过后台分派给运营人员或律师处理。MVP 阶段以图文咨询/留言咨询为主，后续扩展即时聊天、电话咨询、付费咨询和 AI 预问诊。

#### 前台功能

1. 咨询入口：首页、文章详情、法规详情、独立咨询页。
2. 咨询分类：婚姻家事、劳动工伤、交通事故、合同纠纷、房产纠纷、公司法务、刑事案件、知识产权、债权债务、其他。
3. 咨询表单：
   - 问题标题
   - 问题描述
   - 问题分类
   - 所在地区
   - 联系方式
   - 附件上传，MVP 可选
   - 是否公开展示，MVP 可选
4. 咨询记录：用户登录后查看自己的咨询列表、回复状态和详情。
5. 咨询详情：展示问题、回复、处理状态、时间线。
6. 咨询提醒：站内消息，后续扩展短信/邮件。
7. 合规提示：提交前提示平台回复仅供参考，正式法律意见需结合材料由执业律师出具。

#### 后台功能

1. 咨询列表：按状态、分类、地区、关键词、提交时间筛选。
2. 咨询状态：待处理、处理中、已回复、需补充、已关闭、无效。
3. 咨询分派：分派给运营人员或律师。
4. 咨询回复：支持文字回复，后续支持附件和内部备注。
5. 敏感信息处理：公开展示前可脱敏姓名、电话、身份证号等信息。
6. 标签沉淀：可将高质量咨询转为 FAQ 或案例解析。

#### 验收标准

1. 游客可以提交咨询，登录用户可以查看咨询记录。
2. 后台可以筛选、分派、回复、关闭咨询。
3. 咨询提交后有明确状态。
4. 前台展示法律免责声明。

### 7.4 法律法规查询

#### 功能说明

提供法律法规检索、分类浏览和详情查看能力。MVP 可先维护核心法规数据，后续接入权威数据源或建立同步机制。

#### 前台功能

1. 法规搜索：支持标题和正文关键词。
2. 条件筛选：
   - 法规类型：法律、行政法规、司法解释、部门规章、地方性法规、政策文件。
   - 效力状态：现行有效、已修改、已废止、尚未生效。
   - 发布机关
   - 发布日期
   - 施行日期
   - 地区
3. 法规列表：标题、发布机关、发布日期、效力状态、摘要。
4. 法规详情：
   - 标题
   - 文号
   - 发布机关
   - 发布日期
   - 施行日期
   - 效力状态
   - 正文
   - 目录锚点
   - 来源链接
5. 关联内容：相关法律知识、相关新闻、相关咨询、相关案例。
6. 收藏功能：登录用户可收藏法规。
7. 高亮搜索词。

#### 后台功能

1. 法规新增、编辑、发布、下线。
2. 法规导入：支持 Excel/CSV 导入，后续支持批量同步。
3. 法规版本管理：记录修改前后版本，MVP 可先记录更新时间。
4. 效力状态维护。
5. 来源链接维护。

#### 验收标准

1. 用户可按关键词搜索法规。
2. 用户可按类型、效力状态、发布机关筛选。
3. 法规详情页字段完整。
4. 后台可维护法规数据。

### 7.5 法律知识库

#### 功能说明

用通俗语言解释常见法律问题，承接搜索流量，并引导用户咨询。

#### 功能点

1. 知识分类：婚姻、劳动、交通、合同、房产、公司、刑事、知识产权、债务、消费者权益。
2. 知识文章：问题型标题、摘要、正文、适用场景、相关法条、相关咨询。
3. FAQ：按问题聚合短答案。
4. 专题页：如“劳动合同解除专题”“离婚财产分割专题”“交通事故赔偿专题”。
5. 热门标签：支持按标签聚合。

#### 验收标准

1. 用户可按分类阅读知识文章。
2. 知识详情页可跳转到相关法规和在线咨询。
3. 后台可维护知识文章和 FAQ。

### 7.6 案例解析

#### 功能说明

展示典型案例和裁判观点，用于帮助用户理解类似问题。

#### 功能点

1. 案例列表：按案由、领域、法院层级、发布时间筛选。
2. 案例详情：基本案情、争议焦点、裁判结果、法律依据、实务提示。
3. 关联法规和知识文章。
4. 免责声明：案例仅供参考，不代表相同结果。

#### 验收标准

1. 用户可浏览案例解析。
2. 案例详情结构化展示。
3. 后台可新增和编辑案例。

### 7.7 文书模板

#### 功能说明

提供常用法律文书模板，提升实用性和转化率。

#### 功能点

1. 模板分类：合同协议、劳动人事、婚姻家事、诉讼仲裁、公司治理、知识产权。
2. 模板详情：适用场景、注意事项、预览内容、下载入口。
3. 下载权限：MVP 可免费下载，后续支持登录下载、积分/会员下载。
4. 模板关联咨询：用户下载前后可发起咨询。

#### 验收标准

1. 用户可查看模板列表和详情。
2. 后台可上传和管理模板文件。

### 7.8 找律师/找机构

#### 功能说明

为后续法律服务转化做准备。MVP 可以先做展示和咨询分派，不做复杂入驻。

#### 功能点

1. 律师列表：姓名、头像、执业机构、执业年限、擅长领域、地区。
2. 律师详情：简介、执业证号、服务领域、文章/案例、咨询入口。
3. 机构列表：律所、法律服务机构、公证/调解/法援机构，后续扩展。
4. 筛选条件：地区、领域、经验、服务方式。
5. 后台资质审核：律师资料需审核后展示。

#### 验收标准

1. 用户可按领域和地区查看律师。
2. 后台可维护律师和机构信息。

### 7.9 企业法务专区

#### 功能说明

面向中小企业提供合规内容、法规专题和咨询入口。

#### 功能点

1. 企业合规专题：劳动用工、合同管理、股权治理、知识产权、数据合规、税务风险。
2. 政策法规推荐：按企业经营场景聚合。
3. 企业咨询表单：支持公司名称、行业、问题类型、联系人。
4. 企业服务介绍：常年法律顾问、合同审查、劳动合规、专项咨询，MVP 可作为展示页面。

#### 验收标准

1. 用户可以查看企业法务专题。
2. 企业用户可以提交企业咨询。

### 7.10 全站搜索

#### 功能说明

统一检索新闻、法规、知识、案例、模板和律师。

#### 功能点

1. 搜索框：支持关键词输入。
2. 搜索结果分类 Tab：全部、新闻、法规、知识、案例、模板、律师。
3. 搜索高亮。
4. 热门搜索词。
5. 无结果推荐：展示相关分类和咨询入口。

#### 验收标准

1. 用户可以从导航和首页发起搜索。
2. 搜索结果可按内容类型切换。

## 8. 核心业务流程

### 8.1 用户提交咨询流程

1. 用户进入在线咨询页。
2. 用户选择咨询分类。
3. 用户填写问题描述和联系方式。
4. 系统创建咨询单，状态为待处理。
5. 后台运营人员查看咨询。
6. 运营人员回复或分派律师。
7. 律师/运营人员回复咨询。
8. 用户查看回复。
9. 咨询关闭或继续补充。

### 8.2 新闻发布流程

1. 运营人员创建新闻草稿。
2. 编辑标题、封面、摘要、正文、分类和标签。
3. 提交审核。
4. 审核通过后发布。
5. 前台展示。
6. 后续可下线或更新。

### 8.3 法规查询流程

1. 用户输入关键词。
2. 系统返回法规列表。
3. 用户使用筛选条件缩小范围。
4. 用户进入法规详情。
5. 用户查看正文、效力状态、来源和关联内容。
6. 用户收藏或发起相关咨询。

## 11. 非功能需求

### 11.1 性能

1. 首页首屏加载时间建议小于 2 秒。
2. 新闻和法规列表接口响应时间建议小于 500ms。
3. 法规搜索在 10 万条以内数据量时应保持可用，后续可接入 Elasticsearch 或 Meilisearch。

### 11.2 安全

1. 用户密码必须哈希存储。
2. 后台接口必须鉴权。
3. 管理系统需要 RBAC 权限控制。
4. 用户联系方式、咨询内容等敏感数据需要限制访问。
5. 文件上传需要限制类型、大小，并进行安全校验。
6. 前台表单需要防刷机制，后续可接入验证码。

### 11.3 合规

1. 法律内容需展示来源和发布时间。
2. 在线咨询回复应保留免责声明。
3. 律师展示应审核执业信息。
4. 公开咨询内容前应脱敏个人信息。
5. 法规数据采集和展示需注意来源授权、引用规范和更新机制。

### 11.4 SEO

1. 新闻、法规、知识、案例详情页需要独立 SEO 标题和描述。
2. 支持 sitemap.xml，后续扩展。
3. 支持 robots.txt。
4. URL 结构尽量语义化。

### 11.5 可维护性

1. 内容分类、标签、首页推荐位应后台可配置。
2. 前后端接口使用统一错误码和响应格式。
3. 关键数据需要创建时间、更新时间、操作人。

## 16. 风险与待确认问题

### 16.1 风险

1. 法规数据来源和更新成本较高，需要明确数据维护方式。
2. 在线咨询涉及法律服务边界，需要合规声明和回复审核机制。
3. 如果开放律师入驻，需要资质审核和服务质量管理。
4. 法律内容专业性要求高，需要编辑审核流程。
5. 法规全文搜索后期可能需要专业搜索引擎支撑。

### 16.2 待确认问题

1. 网站主要面向中国大陆用户，还是也覆盖港澳台/海外法律内容？
2. MVP 阶段是否需要用户登录后才能提交咨询？
3. 咨询回复由平台运营人员处理，还是一开始就引入律师角色？
4. 法规数据是手动维护、采集公开来源，还是采购/接入第三方法律数据库？
5. 是否计划做付费咨询和律师入驻？
6. 是否需要移动端 H5 优先适配？
7. 是否需要企业客户专属入口和企业后台？
